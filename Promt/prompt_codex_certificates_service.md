# Prompt для Codex: AI-сервис базы сертификатов металлопроката

Ты — senior full-stack разработчик и AI engineer.
Создай production-ready веб-сервис для хранения, распознавания и удобного поиска сертификатов продукции.

---

## 1. Цель проекта

Нужен сервис, куда пользователь загружает сертификаты товара в разных форматах:

- PDF
- JPG
- JPEG
- PNG
- WEBP
- TIFF
- другие популярные форматы изображений и документов

После загрузки сервис должен автоматически:

1. Распознать текст из файла.
2. Определить, что это за товарная позиция.
3. Найти и сохранить ключевые данные сертификата.
4. Сформировать базу сертификатов.
5. Дать удобный поиск по базе.

---

## 2. Что нужно извлекать из сертификата

Сервис должен определять и сохранять:

- наименование товара;
- тип продукции;
- размер;
- марку стали / материал;
- ГОСТ / ТУ;
- дату сертификата;
- номер сертификата;
- производителя;
- поставщика, если указан;
- номер партии / плавки, если указан;
- исходный файл;
- распознанный текст;
- уверенность OCR-распознавания.

---

## 3. Пример распознавания

Если в сертификате есть строка:

```text
Труба профильная 120х120х4 Ст3 ГОСТ 8639-82
```

Система должна сохранить данные так:

```json
{
  "product_type": "Труба профильная",
  "dimensions": "120х120х4",
  "material": "Ст3",
  "gost": "ГОСТ 8639-82",
  "normalized_product_name": "Труба профильная 120х120х4 Ст3 ГОСТ 8639-82"
}
```

Дополнительные примеры позиций:

- Лист 3 мм 09Г2С ГОСТ 19903
- Арматура А500С 12 мм
- Швеллер 16П Ст3сп
- Уголок 50х50х5 С255
- Круг 20 Ст45
- Балка 20Б1 Ст3
- Труба ВГП 25х3,2 ГОСТ 3262-75
- Труба электросварная 76х3,5 ГОСТ 10704-91

## 4. Основной функционал

### 4.1 Загрузка файлов

Сделай страницу загрузки с drag-and-drop.

Функции:

- одиночная загрузка;
- массовая загрузка;
- загрузка PDF и изображений;
- отображение статуса обработки;
- отображение ошибок;
- создание превью документа;
- возможность открыть оригинальный файл.

После загрузки файл должен сохраняться, а обработка должна запускаться в фоне.

---

### 4.2 OCR и извлечение текста

Сделай OCR pipeline.

Можно использовать:

- Tesseract OCR;
- PaddleOCR;
- pdfplumber;
- PyMuPDF;
- OpenCV для предобработки изображений.

OCR должен:

- извлекать текст из PDF;
- извлекать текст из изображений;
- чистить мусорные символы;
- исправлять типичные OCR-ошибки;
- нормализовать русские и латинские символы;
- сохранять исходный распознанный текст;
- сохранять confidence score.

---

### 4.3 AI-парсер сертификатов

Создай отдельный parser engine для металлопроката.

Парсер должен уметь находить:

- тип продукции: труба, лист, арматура, швеллер, уголок, круг, балка и т.д.;
- размеры: 120х120х4, 50х50х5, 12 мм, 3 мм и т.д.;
- марку стали: Ст3, Ст3сп, 09Г2С, С255, Ст45, А500С и т.д.;
- ГОСТ: ГОСТ 8639-82, ГОСТ 10704-91, ГОСТ 19903 и т.д.;
- дату сертификата;
- номер сертификата;
- производителя;
- номер партии / плавки.

Используй комбинированный подход:

- регулярные выражения;
- словари синонимов;
- нормализацию текста;
- fuzzy matching;
- embeddings / semantic search;
- возможность подключить LLM-парсинг в будущем.

Архитектура парсера должна быть расширяемой.

## 6. Поиск

Сделай удобный и быстрый поиск по базе сертификатов.

Поиск должен работать по:

- названию товара;
- размеру;
- марке стали;
- ГОСТ;
- номеру сертификата;
- дате сертификата;
- производителю;
- номеру партии;
- полному тексту сертификата.

Примеры поисковых запросов:

```text
120х120
Ст3
8639
09Г2С
труба профильная
А500С 12
ГОСТ 10704
```

Добавь:

- full-text search PostgreSQL;
- fuzzy search;
- semantic search;
- фильтры;
- сортировку;
- пагинацию;
- подсветку найденных совпадений.

Важно: запрос `труба 120` должен находить `Труба профильная 120х120х4`.

## 9. Обработка в фоне

После загрузки файла задача должна уходить в background worker.

Статусы обработки:

- uploaded;
- processing;
- parsed;
- needs_review;
- failed.

Worker должен:

1. Получить файл.
2. Определить тип файла.
3. Извлечь текст.
4. Запустить OCR при необходимости.
5. Запустить parser engine.
6. Сохранить результат в PostgreSQL.
7. Обновить статус.
